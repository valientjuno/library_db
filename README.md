# library_db
This repository contains  a SQL database in which it's data is randomized. 
How to Run

Create a new database named library_db

Execute the SQL script to create tables

Run the provided INSERT statements to populate sample data

Execute SELECT queries to analyze and explore the data


Technologies Used

SQL

PostgreSQL / SQLite (compatible)



Database Schema

The database includes the following core tables:

authors – Stores author information

books – Stores book details and links to authors

members – Stores library member information

loans – Tracks borrowed books and return status

Relationships:

Each book is associated with an author

Each loan links a member to a borrowed book

















What This Project Demonstrates

Relational database design

Use of primary and foreign keys

Data integrity and normalization

Writing SQL queries using joins and aggregate functions

Preparing a database for backend application integration
